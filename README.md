# own-redis

`own-redis` — это высокопроизводительное хранилище данных типа «ключ-значение», работающее в оперативной памяти (RAM). Проект имитирует базовую функциональность Redis, используя протокол **UDP** для обеспечения минимальных задержек при передаче данных.

##  Особенности
* **In-Memory Storage**: Молниеносный доступ к данным с временной сложностью O(1).
* **UDP Protocol**: Взаимодействие между клиентом и сервером через легковесные UDP-пакеты.
* **Thread-Safety**: Использование примитивов синхронизации из пакета `sync` для предотвращения состояний гонки (Data Races) при конкурентных запросах.
* **TTL (Time-To-Live)**: Поддержка опции `PX` для установки времени жизни ключа в миллисекундах.

##  Технологический стек
* **Language**: Go (только встроенные пакеты).
* **Formatting**: Соответствует стандартам `gofumpt`.
* **Networking**: Пакет `net` (UDP).

## Поддерживаемые команды

| Команда | Описание | Пример |
| :--- | :--- | :--- |
| **PING** | Проверка доступности. Возвращает `PONG`. | `PING` |
| **SET** | Сохранение ключа и значения. Поддерживает `PX`. | `SET user:1 "Alice" PX 5000` |
| **GET** | Получение значения. Возвращает `(nil)`, если ключ не найден. | `GET user:1` |

## Установка и запуск

**1. Сборка проекта:**
```bash
go build -o own-redis .
```

**2. Запуск сервера** (порт по умолчанию 8080):
```bash
./own-redis --port 8080
```

**3. Использование:**
Для взаимодействия с сервером используйте `netcat` в режиме UDP (`-u`):
```bash
# Записать данные
echo "SET foo bar" | nc -u -w 1 0.0.0.0 8080

# Прочитать данные
echo "GET foo" | nc -u -w 1 0.0.0.0 8080
```

## Критерии корректности
* **Case-Insensitivity**: Команды (`PING`, `ping`, `PiNg`) обрабатываются одинаково.
* **Validation**: Строгая проверка аргументов. При ошибке возвращается: `(error) ERR wrong number of arguments for 'SET' command`.
* **Concurrency**: Безопасная работа через `sync.RWMutex`.
* **Zero Panics**: Надежная обработка исключает `nil-pointer dereference` и `index out of range`.
